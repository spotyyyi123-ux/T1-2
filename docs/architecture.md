# Архитектура пайплайна ContractAI

## Общая схема обработки договора

1. **Загрузка файла** — пользователь загружает DOCX или PDF через веб-форму.
2. **Парсинг документа** — извлечение сырого текста (`python-docx` для DOCX, `pypdf` для PDF).
3. **Извлечение сущностей** — RegEx и/или Natasha находят номер договора, дату, сумму, стороны.
4. **Валидация полей** — сверка извлечённых значений с эталонной разметкой (`ground_truth`).
5. **Запись в базу данных** — сохранение документа и извлечённых полей в SQLite.
6. **Реестр (frontend)** — веб-интерфейс для поиска и просмотра каталогизированных договоров.

## Схема базы данных

- `documents` — один договор (или его файл-версия docx/pdf) = одна строка.
- `extracted_fields` — результаты работы алгоритмов извлечения (regex/Natasha), с указанием метода и уверенности.
- `parties` — стороны договора (заказчик/исполнитель), с ИНН/ОГРН.
- `ground_truth` — эталонные значения полей для оценки точности алгоритмов.

Связь: одна запись в `documents` → много записей в `extracted_fields`, `parties`, `ground_truth`
(поле `document_id` во всех трёх таблицах ссылается на `documents.id`).
